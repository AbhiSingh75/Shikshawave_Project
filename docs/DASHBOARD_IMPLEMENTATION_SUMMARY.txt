================================================================================
FEATURES IMPLEMENTED
================================================================================

1. STORED PROCEDURE (Proc_Dashboard_Students_Get)
   Parameters:
   - @SchoolID (INT) - Filter by school
   - @ClassID (INT) - Filter by class
   - @SectionID (INT) - Filter by section
   - @AcademicYear (NVARCHAR) - Filter by academic year
   - @Gender (NVARCHAR) - Filter by gender
   - @Category (NVARCHAR) - Filter by category
   - @FromDate (DATE) - Filter by created date from
   - @ToDate (DATE) - Filter by created date to
   - @ShowActiveOnly (BIT) - Show only active students (default: 1)

   Returns:
   - Summary statistics (total, male, female, categories, active/inactive)
   - Class-wise breakdown with gender distribution

2. DASHBOARD FILTERS (India Standard)
   ✓ From Date - Filter by student creation date (from)
   ✓ To Date - Filter by student creation date (to)
   ✓ Class dropdown (1st to 12th)
   ✓ Section dropdown (A, B, C, etc.)
   ✓ Gender filter (Male/Female)
   ✓ Category filter (General/OBC/SC/ST)
   ✓ Academic Year filter
   ✓ Reset button - Clear all filters

3. DASHBOARD CARDS (Visible only for School Admin & Teacher)
   ✓ Active Students - Shows active student count (default)
   ✓ Male Students - Shows male count
   ✓ Female Students - Shows female count
   ✓ Category Breakdown - Shows category distribution

4. DYNAMIC FEATURES
   ✓ Shows only active students by default
   ✓ Date range filtering (From/To Date)
   ✓ Profile-based visibility (Profile ID 2 & 3 only)
   ✓ Reset filters functionality
   ✓ Async filter loading on page load
   ✓ Section dropdown updates based on class selection
   ✓ Real-time card updates without page reload
   ✓ AJAX-based data fetching

================================================================================
API ENDPOINTS
================================================================================

1. GET /api/dashboard/students/
   Authentication: Required
   Query Parameters:
   - from_date (optional) - Format: YYYY-MM-DD
   - to_date (optional) - Format: YYYY-MM-DD
   - class_id (optional)
   - section_id (optional)
   - gender (optional)
   - category (optional)
   - academic_year (optional)
   - show_active_only (optional, default: 1)
   
   Response:
   {
     "success": true,
     "stats": {
       "total_students": 150,
       "male_students": 80,
       "female_students": 70,
       "general_category": 90,
       "obc_category": 40,
       "sc_category": 15,
       "st_category": 5,
       "active_students": 145,
       "inactive_students": 5
     },
     "class_breakdown": [...]
   }

2. Existing APIs Used:
   - GET /api/classes/ - Load class dropdown
   - GET /api/sections/?class_id={id} - Load section dropdown
   - GET /api/academic-years/ - Load academic year dropdown

================================================================================
INSTALLATION STEPS
================================================================================

1. Execute Stored Procedure:
   sqlcmd -S your_server -d your_database -i database/procedures/Proc_Dashboard_Students_Get.sql

2. Restart Django Server:
   python manage.py runserver

3. Access Dashboard:
   - Login to system
   - Navigate to /dashboard/
   - Use filters to view student statistics

================================================================================
USAGE EXAMPLE
================================================================================

1. View All Students:
   - Leave all filters as "All"
   - Click "Apply Filters"

2. View Male Students in Class 5:
   - Select "Class 5" from Class dropdown
   - Select "Male" from Gender dropdown
   - Click "Apply Filters"

3. View OBC Category Students:
   - Select "OBC" from Category dropdown
   - Click "Apply Filters"

4. View Specific Section:
   - Select Class first
   - Select Section from dropdown
   - Click "Apply Filters"
