================================================================================
STORED PROCEDURE ACTIONS
================================================================================

Action: INSERT
- Validates required fields
- Prevents duplicate component names per school
- Validates component type (Earning/Deduction)
- Returns JSON response

Action: UPDATE
- Validates component exists
- Prevents duplicate names
- Validates component type
- Returns JSON response

Action: DELETE
- Soft deletes (sets IsDeleted = 1)
- Preserves data
- Returns JSON response

Action: RESTORE
- Restores deleted component (sets IsDeleted = 0)
- Clears DeletedBy and DeletedAt
- Returns JSON response

================================================================================
ACCESS CONTROL
================================================================================

Super Admin (ProfileID = 1):
- Can view all schools' salary components
- Can manage components for any school

School Admin (ProfileID = 2):
- Can view only their school's components
- Can manage only their school's components

Other Profiles:
- Access controlled via ProfileMenuMapping table

================================================================================
COMPONENT TYPES
================================================================================

Earning:
- Components that add to salary
- Examples: Basic Salary, HRA, DA, Allowances, Bonus
- Badge Color: Green

Deduction:
- Components that subtract from salary
- Examples: PF, ESI, Tax, Loan, Advance
- Badge Color: Red

================================================================================
VALIDATION RULES
================================================================================

1. Component Name: Required, max 100 characters
2. Component Type: Must be 'Earning' or 'Deduction'
3. School ID: Required
4. Duplicate Check: No duplicate names per school
5. Soft Delete: Preserves data (IsDeleted = 1)
6. Restore: Available for deleted components
